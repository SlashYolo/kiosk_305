# МАИ.Стенд — Информационный киоск кафедры 305

Сенсорный TV-стенд на базе Lubuntu (LXQt / Openbox) + Chromium в режиме киоска.  
SPA-приложение на чистом HTML/CSS/JS, Flask-прокси для API, офлайн-кэш.

## Лицензия

[GNU AGPL v3](LICENSE) — подробности в файле `LICENSE`.

## Быстрый старт

```bash
# Клонируем
git clone <repo-url> ~/mai-kiosk && cd ~/mai-kiosk

# Устанавливаем зависимости и настраиваем автозапуск
```

## Учебные лаборатории

Карточки лабораторий кликабельны → открывается модалка с подробной информацией и каруселью фото.

**Где править контент:** массив `LABS` в `mai_kiosk.html`. Каждый объект содержит:
- `icon`, `title`, `short`, `room` — данные карточки
- `description` — длинное описание (модалка)
- `equipment` — список оборудования
- `activities` — чем занимаются
- `photos` — пути к фото (например `/lab-photos/3-101-1.jpg`)

**Фото:** папка `lab-photos/` рядом с `server.py`. Если файла нет → автоматическая заглушка «📷 Фото скоро будет добавлено».

**Карусель:** автопрокрутка каждые 6 сек, стрелки, точки, свайп на тачскрине. Если фото одно — навигация скрывается.

## Фотографии преподавателей

Объект `TEACHER_PHOTOS` в `mai_kiosk.html` маппит GUID преподавателя → имя файла. Все 22 преподавателя кафедры уже прописаны с именами файлов типа `aleshin.jpg`, `afonin.jpg` и т.д.

**Формат аватарок:** вертикальные прямоугольники 52×68px с закруглёнными углами (border-radius:14px). Если фото нет → инициалы на цветном фоне в том же формате.

**Фото кладутся в** `avatars_teachers/`. Рекомендуемый размер: не менее 200×260px, портретная ориентация, JPEG/PNG/WebP.

## AFK-видео заставка

Положи файл `on.mp4` в папку проекта. После **1 минуты** бездействия страница показывает видео на весь экран с плавным fade-in. Любой тач/клик/клавиша → заставка плавно исчезает.

Заставка работает **в браузере** через HTML5 `<video>` — зависимости `xprintidle` / `mpv` больше не нужны.

**Параметры** (в `mai_kiosk.html`):
- `AFK_IDLE_MS` — порог простоя в мс (по умолчанию `60000` = 1 мин)
- `AFK_CHECK_MS` — частота проверки (по умолчанию `1000` = 1 сек)

**Рекомендации к видео:** H.264, MP4, 720p/1080p, 24–30 fps, без звука. Первый и последний кадры одинаковые для бесшовного looping.

## Кэширование расписания

`cache_schedule.py` полностью кэширует расписание МАИ + публикации преподавателей на диск.

```bash
# Последовательный режим
python3 cache_schedule.py

# Параллельный (рекомендуется, в 5–10 раз быстрее)
python3 cache_schedule.py --parallel 5

# Стартап-режим (из start.sh / mai-kiosk.sh)
python3 cache_schedule.py --startup --parallel 5
```

**Инкрементальный режим:** при каждом запуске считается SHA1-хэш ответа (без служебных timestamp-полей). Если ответ не изменился — файл не перезаписывается, в логе видно как `skipped`. Экономит время при повторных запусках.

**Прогресс:** пишется в `schedule_cache/_progress.json`. Фронтенд опрашивает `/api/cache-progress` раз в 3 секунды и показывает ненавязчивую плашку в правом нижнем углу: фазу (группы/преподаватели/публикации), процент, и сколько пропущено без изменений.

**Автообновление:** cron каждый день в 10:00:
```bash
0 10 * * * python3 ~/mai-kiosk/wifi_schedule_update.py >> ~/mai-kiosk/update.log 2>&1
```

## Wi-Fi management

`wifi_schedule_update.py` управляет подключением Wi-Fi для обновления кэша (актуально когда стенд стоит без постоянного интернета).

```bash
# Первичная настройка (один раз, шифрует SSID/пароль на machine-id)
python3 wifi_schedule_update.py --setup

# Старт стенда: подключиться → кэш → отключиться (фоновый вызов из start.sh)
python3 wifi_schedule_update.py --startup

# Плановое обновление (cron в 10:00)
python3 wifi_schedule_update.py
```

Если интернет уже есть — Wi-Fi не трогается, кэш запускается напрямую.

## Онлайн / офлайн режим

Фронтенд пингует `/api/health` каждые 5 секунд:
- **Сеть есть** → данные берутся онлайн через прокси
- **Сеть пропала на 5+ сек** → переход на кэш, красная плашка «📦 Из кэша»
- **Сеть вернулась** → через 10 сек возвращается в онлайн

Плашка и бейдж на карточке расписания отражают текущий источник данных.

## Безопасность киоска (lockdown)

На уровне **JavaScript** заблокированы: внешние ссылки, `window.open`, `Ctrl+T/N/W/L`, `F11/F12`, перетаскивание файлов, контекстное меню.

На уровне **Chromium** (`mai-kiosk.sh`): `--kiosk --block-new-web-contents --disable-extensions --disable-translate --disable-pinch --disable-sync`.

На уровне **ОС** (`hide-panel.sh` + `install.sh`): скрытие панели LXQt, блокировка `Alt+F4 / Super / Ctrl+Alt+T` через xbindkeys.

## API сервера

| Эндпоинт | Описание |
|---|---|
| `GET /` | SPA (mai_kiosk.html) |
| `GET /on.mp4` | AFK-видео (Range-поддержка для looping) |
| `GET /lab-photos/<file>` | Фотографии лабораторий |
| `GET /avatars_teachers/<file>` | Аватарки преподавателей |
| `GET /api/health` | Healthcheck (пингуется фронтом) |
| `GET /api/cache-progress` | Прогресс обновления кэша |
| `GET /api/groups` | Список групп МАИ |
| `GET /api/teachers` | Список преподавателей |
| `GET /api/schedule/<name>` | Расписание группы / преподавателя / аудитории |
| `GET /api/teacher-pubs/<guid>` | Публикации преподавателя |
| `GET /api/cache-status` | Статистика кэша |
| `GET /api/cache-names?type=group\|teacher` | Имена из офлайн-кэша |
| `POST /api/show-panel` | Показать панель задач LXQt (для выхода из киоска) |
| `POST /api/fullscreen-exit` | Legacy: xdotool F11 (не работает в --kiosk) |
